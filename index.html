Memahami dasar dasar jaringan komputer

Model OSI adalah kerangka konseptual yang digunakan untuk memahami cara kerja jaringan komputer. Model ini dibagi menjadi 7 lapisan , di mana setiap lapisan memiliki fungsi spesifik dalam proses komunikasi data antar perangkat.

Apa itu Model OSI?

OSI adalah singkatan dari Open Systems Interconnection . Ini adalah model referensi yang dikembangkan oleh ISO (International Organization for Standardization) pada tahun 1984. Tujuannya adalah untuk menyediakan standar global agar sistem dari vendor yang berbeda dapat berkomunikasi satu sama lain. Kenapa Penting? Standarisasi : Memastikan bahwa perangkat keras dan perangkat lunak dari vendor yang berbeda dapat bekerja bersama. Modularitas : Setiap lapisan fokus pada tugas tertentu, sehingga lebih mudah untuk mengelola dan memperbaiki masalah. Troubleshooting : Jika terjadi masalah, kita bisa mengidentifikasi lapisan mana yang bermasalah.

Mengapa Model OSI Penting

Interoperabilitas : Memungkinkan perangkat dari vendor yang berbeda untuk berkomunikasi tanpa hambatan. Misalnya, laptop Windows dapat terhubung ke printer dari vendor lain. Modularitas : Setiap lapisan memiliki fungsi spesifik, sehingga pengembangan teknologi baru tidak harus mengubah seluruh sistem. Contohnya, jika ada inovasi di lapisan fisik (misalnya, kabel baru), lapisan lain tetap tidak terpengaruh. Troubleshooting : Jika terjadi masalah, kita bisa langsung mengecek lapisan mana yang bermasalah. Misalnya, jika data tidak sampai ke tujuan, mungkin ada masalah di Lapisan Jaringan atau Lapisan Transport .

7 Lapisan Model OSI

Sports bring people together and teach discipline, Slide 4: Penjelasan Umum 7 Lapisan OSI Judul : "7 Lapisan Model OSI" Penjelasan : Model OSI dibagi menjadi 7 lapisan , dimulai dari lapisan terbawah (fisik) hingga lapisan tertinggi (aplikasi). Berikut adalah gambaran umum dari setiap lapisan: Lapisan Fisik (Physical Layer) Bertanggung jawab atas transmisi data dalam bentuk sinyal fisik (listrik, optik, atau radio). Contoh: Kabel Ethernet, Wi-Fi, Bluetooth. Lapisan Data Link (Data Link Layer) Mengatur transfer data antar perangkat dalam jaringan lokal. Mendeteksi dan mengoreksi kesalahan. Contoh: Ethernet, PPP (Point-to-Point Protocol). Lapisan Jaringan (Network Layer) Menentukan rute pengiriman data dari sumber ke tujuan. Bertanggung jawab atas alamat IP dan routing. Contoh: IP (Internet Protocol), ICMP. Lapisan Transport (Transport Layer) Memastikan data dikirim dengan benar dan lengkap. Mengontrol aliran data dan mengatasi kesalahan. Contoh: TCP (Transmission Control Protocol), UDP (User Datagram Protocol). Lapisan Sesi (Session Layer) Mengelola sesi komunikasi antar aplikasi. Bertanggung jawab untuk membuka, memelihara, dan menutup sesi. Contoh: Protokol seperti NetBIOS, RPC. Lapisan Presentasi (Presentation Layer) Mengubah data agar dapat dipahami oleh aplikasi penerima. Bertanggung jawab atas enkripsi, kompresi, dan translasi data. Contoh: JPEG, GIF, SSL/TLS. Lapisan Aplikasi (Application Layer) Berinteraksi langsung dengan pengguna. Menyediakan layanan seperti email, web browsing, dan file transfer. Contoh: HTTP, FTP, SMTP.

Lapisan Fisik (Physical Layer)

Fungsi Utama : Mengirimkan data dalam bentuk sinyal fisik (listrik, optik, atau gelombang radio). Bertanggung jawab atas media transmisi seperti kabel, fiber optik, dan sinyal nirkabel. Contoh Media Transmisi : Kabel : Kabel tembaga (Ethernet), fiber optik. Nirkabel : Wi-Fi, Bluetooth, sinyal radio. Protokol Terkait : Ethernet, DSL, USB.

Lapisan Data Link (Data Link Layer)

Fungsi Utama : Mengatur transfer data antar perangkat dalam jaringan lokal (LAN). Mendeteksi dan mengoreksi kesalahan dalam transmisi data. Dua Sub-Lapisan : LLC (Logical Link Control) : Mengelola komunikasi antar perangkat. MAC (Media Access Control) : Mengontrol akses ke media transmisi. Contoh Protokol : Ethernet, PPP (Point-to-Point Protocol), Wi-Fi (IEEE 802.11)dan masig banyak data lainnya.
